How HireAI Works

Built for the AWS Hackathon — here's what makes this system different

◆

AI-Powered Recruitment Screening

HireAI uses Amazon Bedrock to semantically analyse resumes against a job description — not just keyword matching. It understands context, infers skill levels, and ranks candidates by true fit.

🎯

What Sets Us Apart

  • Semantic understanding via Amazon Bedrock — reads intent, not just words
  • Natural language job descriptions — no rigid form filling
  • Unbiased by design — names, gender, and age are stripped before scoring
  • Scores every candidate with a transparent AI reasoning summary
  • Works on bulk CSV or individual PDF resumes
📋

Duplicate Detection

  • Fingerprints each resume using a hash of normalised content
  • Detects re-submissions with minor edits (extra spaces, punctuation swaps)
  • Flags near-duplicates using similarity scoring — not just exact matches
  • Only the highest-scoring version of a duplicate is kept in the ranked list
  • Duplicate count is surfaced in the results so reviewers stay informed
✏️

Whitespace & Formatting Abuse

  • All text is normalised before analysis — excess whitespace, tabs, and line breaks are collapsed
  • Hidden text tricks (white-on-white keywords) are neutralised at parse time
  • PDF text extraction strips formatting metadata, leaving only readable content
  • Scores are based on semantic meaning, so padding with irrelevant keywords doesn't help
🔐

Anti-Cheating Measures

  • Keyword stuffing is penalised — Bedrock detects unnatural skill density
  • Experience claims are cross-validated against described responsibilities
  • Suspiciously high skill counts relative to experience years are flagged
  • Each candidate gets a confidence score alongside their AI score
  • Reviewers see a plain-English reason for every score — nothing is a black box
☁️

AWS Architecture

  • Amazon Bedrock — LLM inference for resume analysis
  • AWS Lambda — serverless processing per resume
  • Amazon S3 — resume file storage
  • Amazon DynamoDB — candidate records and scores
  • All processing is stateless and scales to thousands of resumes
📊

Scoring Methodology

  • Skills match (40%) — how well skills align with the job description
  • Experience relevance (30%) — years and domain match
  • Responsibility depth (20%) — quality of described work, not just titles
  • Consistency (10%) — coherence between claimed skills and actual experience
  • Final score is 0–100, with a written justification per candidate
